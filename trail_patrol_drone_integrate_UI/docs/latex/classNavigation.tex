\doxysubsection{Detailed Description}
Main navigation controller class for drone flight control. 

This class manages low-\/level drone navigation including\+:
\begin{DoxyItemize}
\item \mbox{\hyperlink{structWaypoint}{Waypoint}} following with state machine control
\item Three-\/tier obstacle detection system\+:
\begin{DoxyItemize}
\item Emergency\+: $<$ 0.\+5m (immediate fly-\/over)
\item Normal\+: $<$ 2.\+0m (scan and choose avoidance)
\item Avoidance\+: $<$ 1.\+5m (during avoidance maneuvers)
\end{DoxyItemize}
\item Left/right path scanning for fly-\/around decisions
\item Velocity command generation (linear and angular)
\item UI control integration (pause, resume, emergency stop)
\end{DoxyItemize}

The navigation node receives waypoints from the mission node and publishes cmd\+\_\+vel commands to control the drone\textquotesingle{}s movement.

Processes laser scan data for multi-\/level obstacle detection. 

Implements a three-\/tier obstacle detection system\+:
\begin{DoxyEnumerate}
\item Emergency detection ($<$ 0.\+5m)\+: Always active, triggers immediate fly-\/over
\item Normal detection ($<$ 2.\+0m)\+: Active when MOVING and aligned, triggers 1s scan

Scans left and right sides to determine best avoidance direction. 

After obstacle detection triggers a 1-\/second scan, this function analyzes 30° segments on the left (90°) and right (270°) sides of the drone to determine which direction has a clear path.

Avoidance decision logic\+:
\begin{DoxyItemize}
\item Right clear → Direction 1 (fly-\/around right)
\item Left clear → Direction 2 (fly-\/around left)
\item Neither clear → Direction 0 (fly-\/over)
\end{DoxyItemize}

A segment is considered \char`\"{}clear\char`\"{} if 80\% of laser points are beyond 3.\+0m.
